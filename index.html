Origin / operating model

I got into security the way a lot of people do: by breaking something I wasn’t supposed to, panicking a little, and then reading way too much about how it worked. These days I do it on purpose, with permission, and get paid for it.

My focus is on the bugs that live in application logic rather than in a CVE database — broken access control, authorization bypasses, authentication flaws, and the weird edge cases APIs leave wide open. I spend more time reading JavaScript bundles and API responses than running scanners.

That instinct doesn’t stop at the browser. I pop binaries open in Binary Ninja Ultimate when an API alone doesn’t tell the whole story, and I audit Solidity contracts for DeFi protocols through competitive contests on Code4rena. Different attack surface, same assumption: the logic is wrong until proven otherwise.

When I’m not chasing bounties, I’m writing up what I found (the parts I’m allowed to share), building small recon tools, and trying to get better at explaining impact clearly enough that triagers don’t have to ask twice.

Everything published on this site is for education. If you’re testing any of it yourself, make sure you’re explicitly authorized to.

02 / 04Coverage Matrix

Where the time actually goes.

No two targets are the same, but most of my work falls into one of these buckets — whether it’s the web, a binary, or a smart contract.

01 / 09

Access Control & IDOR

Object-level authorization bugs, BOLA, and horizontal/vertical privilege escalation across web and mobile APIs. The unglamorous class that still pays.

02 / 09

Auth, Session & OAuth

Login flow abuse, JWT attacks, session fixation, and OAuth/SSO misconfigurations that lead quietly to account takeover.

03 / 09

SSRF & Server-Side

Webhooks, URL previews, PDF and image processors — any feature that makes an outbound request on my behalf is worth a closer look.

04 / 09

XSS & Client-Side

Reflected, stored, and DOM XSS; CSP bypasses; the edge cases modern frontend frameworks quietly reintroduce.

05 / 09

API & GraphQL Security

Introspection leaks, batching abuse, excessive data exposure, and missing field-level authorization on schemas that were never meant to be public.

06 / 09

Recon & Automation

Subdomain enumeration, JavaScript analysis, and small pipelines that keep new attack surface flowing in automatically instead of by chance.

07 / 09

Reverse Engineering

Static and dynamic binary analysis in Binary Ninja Ultimate — unpacking, patch diffing, and figuring out what a binary really does when the docs won’t tell you.

08 / 09

Smart Contract & DLT Security

Solidity and EVM logic bugs — reentrancy, access control, and economic exploits — audited through competitive contests on Code4rena.

09 / 09

Mobile & Native AppSec

iOS and Android app testing plus native library review — insecure storage, hardcoded secrets, and the native code lurking behind a pretty UI.
